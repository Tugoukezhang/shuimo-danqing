## 一、踩圈切色机制

### 原版怎么做的

| 维度 | 原版实现 |
|------|---------|
| **触发时机** | BOSS战阶段，场地随机刷出"光暗圈" |
| **圈的表现** | 地面出现带颜色的光圈（红/黄/紫/蓝/绿），有脉冲动画 |
| **交互方式** | 玩家角色踩上去 → 当前攻击颜色切换为圈的颜色 |
| **策略意义** | BOSS有弱点颜色，必须踩对应颜色圈→切色→打弱点才能累积击破条 |
| **刷新节奏** | 每隔5-8秒刷出新圈，旧圈5秒后消失 |
| **视觉反馈** | 踩圈时有"颜色注入"动画，角色外观颜色跟随切换 |
| **圈的数量** | 同时场上2-3个不同颜色的圈 |

### 我们要做什么（水墨丹青版）

| 维度 | 水墨丹青实现 |
|------|-------------|
| **名称** | 墨池切色 / 踩墨换色 |
| **触发时机** | 全程战斗（不仅BOSS），从第2波开始出现 |
| **圈的表现** | 地面出现"墨池"——五色水墨晕染圆形，带波纹动画 + 对应颜色名（朱砂/石绿/藤黄/花青/紫） |
| **交互方式** | 玩家踩入墨池 → 当前弹丸颜色切换为该墨池颜色 → 弹丸视觉同步变化 |
| **策略意义** | 精英怪有颜色护甲 → 必须踩对应墨池换色才能高效击破 → BOSS弱点色同理 |
| **刷新节奏** | 每6秒刷出1-2个新墨池，旧墨池8秒后干涸消失（有渐隐动画） |
| **视觉反馈** | 踩入时墨池"溅起"水花粒子，角色外环颜色渐变为新色，弹丸颜色立即切换 |
| **圈的数量** | 同时场上2-3个不同颜色墨池 |
| **墨池位置** | 在玩家周围300-500px范围内随机生成（不能太远够不到） |

### 与原版的差异（我们改进了什么）

| 差异点 | 原版 | 我们 | 理由 |
|--------|------|------|------|
| 出现时机 | 仅BOSS战 | 全程（第2波起） | 让小兵阶段也有策略，不只是无脑突突突 |
| 命名 | 光暗圈 | 墨池 | 水墨主题统一 |
| 消失反馈 | 直接消失 | 干涸渐隐动画 | 更有水墨质感 |
| 位置 | 场地固定位置 | 玩家周围随机 | 大地图上不能太远够不到 |

### 技术实现拆解

```
1. colorPool 对象池
   - 位置(x,y)、颜色(colorIndex)、半径(35px)、存在时间(life)、最大寿命(8s)
   - 状态：forming(0.5s出现) → active → drying(1s干涸) → dead

2. spawnColorPool()
   - 每6秒调用一次
   - 在玩家周围300-500px随机位置生成
   - 随机选择颜色（排除当前弹丸颜色，保证有换的必要）
   - 同时最多3个活跃墨池

3. 踩入检测
   - update()中检测玩家与墨池距离 < 墨池半径
   - 进入时：切换 state.bulletColorIndex = pool.colorIndex
   - 视觉：角色外环渐变 + 墨池溅起水花粒子

4. 绘制 drawColorPool()
   - ctx.translate内（世界坐标）
   - 底层：半透明圆形+颜色填充
   - 动画：波纹扩散（sin周期）
   - 标签：颜色名浮于上方

5. UI提示
   - 墨池出现时，小地图上标记对应颜色圆点
   - 场景边缘箭头指向屏幕外墨池
```

## 二、3人队切换系统

### 原版怎么做的

| 维度 | 原版实现 |
|------|---------|
| **队伍构成** | 3个神明娘组成一队 |
| **选人方式** | 战前选择3人编队 |
| **战斗中切换** | 点击角色头像切换当前操控角色 |
| **切换CD** | 切换后有短暂冷却（约2秒） |
| **切换效果** | 切换时角色入场有短暂无敌帧（约0.3秒） |
| **独立血条** | 每个角色独立HP，退场时HP暂停 |
| **被动效果** | 退场角色被动技能仍然生效（如回血光环） |
| **职业搭配** | 典型搭配：1近战+1射击+1魔法/辅助 |
| **大招QTE** | 切人→释放大招→切下一个人→累积连携（详见QTE部分） |

### 我们要做什么（水墨丹青版）

| 维度 | 水墨丹青实现 |
|------|-------------|
| **队伍构成** | 3个画灵组成一队（从已有3画灵中选） |
| **选人方式** | 画灵选择画面改为选3人小队（依次点选） |
| **战斗中切换** | 屏幕右侧3个画灵头像，点击切换 |
| **切换CD** | 3秒CD（避免无限换人） |
| **切换效果** | 0.3秒无敌帧 + 入场冲击（小范围AOE伤害） |
| **独立血条** | 每个画灵独立HP，退场HP冻结，换人后恢复到冻结值 |
| **被动效果** | 退场画灵的被动仍然生效（青岚回血对全队生效） |
| **角色差异** | 青岚(均衡/回血) / 丹霞(暴击/溅墨) / 墨痕(范围/墨影) |
| **爆发技能** | 每个画灵独立怒气条，切换后使用当前画灵的爆发 |

### 与原版的差异

| 差异点 | 原版 | 我们 | 理由 |
|--------|------|------|------|
| 角色数量 | 大量角色可选 | 目前3选3（全上） | 后续抽卡解锁更多画灵 |
| 职业类型 | 近战/射击/魔法/炮击4种 | 目前只有射击型 | 后续版本加近战/魔法画灵 |
| 切换动画 | 2D角色直接切换 | 水墨消散+凝聚动画 | 更有水墨主题感 |

### 技术实现拆解

```
1. 数据模型
   state.party = [
     { heroId: 0, hp: 100, maxHp: 100, rage: 0, alive: true },  // 当前操控
     { heroId: 1, hp: 80,  maxHp: 100, rage: 0, alive: true },  // 后排1
     { heroId: 2, hp: 60,  maxHp: 100, rage: 0, alive: true },  // 后排2
   ]
   state.activeSlot = 0  // 当前操控的画灵索引

2. 画灵选择画面改造
   - 从"选1个"改为"选3个，依次点选"
   - 选够3人后显示"出发"按钮
   - 队伍预览：显示3人缩略图+属性

3. 战斗中切换 switchHero(slotIndex)
   - 检查CD：switchCd > 0 则不可切换
   - 检查目标画灵是否存活
   - 保存当前画灵状态（hp/rage/位置）
   - 切换activeSlot
   - 加载目标画灵状态
   - 触发0.3秒无敌帧
   - 触发入场冲击（小范围AOE）
   - 重置switchCd = 3秒
   - 播放切换动画（水墨消散+凝聚）

4. 右侧画灵头像UI
   - 3个圆形头像，竖排
   - 当前操控：亮色边框+呼吸动画
   - CD中：灰色遮罩+倒计时
   - 已阵亡：暗红+裂痕
   - 显示HP条和怒气条（小型）

5. 被动效果跨角色
   - 青岚墨愈：退场时仍给全队回血（每5秒+3HP）
   - 丹霞丹心：退场时暴击率仍给全队+5%
   - 墨痕墨守：退场时全队防御+10%

6. 爆发技能独立
   - 每个画灵有独立rage计数
   - 切换后使用当前画灵的爆发技能
   - 爆发类型：青岚回春/丹霞散射/墨痕分身（已有）

7. 死亡处理
   - 画灵HP归0 → 该头像变灰
   - 自动切换到下一个存活的画灵
   - 3个画灵全灭 → 游戏结束
```

## 三、QTE联携系统

### 原版怎么做的

| 维度 | 原版实现 |
|------|---------|
| **触发条件** | 切换角色时短暂出现QTE按键提示 |
| **操作方式** | 在时间窗口内（约1.5秒）点击QTE按钮 |
| **QTE成功** | 触发两人联携攻击——全屏特写动画+高额伤害 |
| **QTE失败** | 不触发联携，正常切换 |
| **连携累积** | 连续QTE成功可累积3人联携大招（最高伤害） |
| **视觉效果** | 全屏特写画面（角色动画演出）+ 屏幕震动 + 大范围染色 |
| **策略意义** | 在BOSS僵直窗口内打出QTE连携 = 最高伤害输出 |

### 我们要做什么（水墨丹青版）

| 维度 | 水墨丹青实现 |
|------|-------------|
| **名称** | 墨韵联携 / 画灵合笔 |
| **触发条件** | 切换画灵时0.8秒窗口内再次点击"合笔"按钮 |
| **操作方式** | 切人后屏幕中央出现"合笔"圆形按钮，0.8秒内点击 |
| **QTE成功** | 两画灵联携攻击——水墨合流特效+大范围AOE+高额伤害 |
| **QTE失败** | 不触发联携，正常切换 |
| **连携累积** | 连续合笔成功 → 2人合笔(普通) → 3人合笔(终极) |
| **视觉效果** | 两色水墨交融→爆炸扩散，3人合笔=五色晕染全屏 |
| **策略意义** | BOSS僵直窗口内打3人合笔 = 终极输出 |

### 与原版的差异

| 差异点 | 原版 | 我们 | 理由 |
|--------|------|------|------|
| QTE窗口 | 1.5秒 | 0.8秒 | 更紧张刺激，不能无脑点 |
| 视觉风格 | 二次元特写 | 水墨交融晕染 | 主题统一 |
| 3人联携 | 有 | 有，但简化为"第三次合笔自动升级" | 不需要额外操作，连续成功自然升级 |

### 技术实现拆解

```
1. QTE触发
   - switchHero()成功后，qteWindow = 0.8秒
   - 屏幕中央出现"合笔"按钮（水墨圆章风格）
   - 每帧倒计时 qteTimer -= dt

2. QTE成功 qteSuccess()
   - qteCombo++（连续成功次数）
   - 根据qteCombo等级：
     - combo=1: 2人合笔（当前+刚退场的）
       → 两色墨流交汇 → 中等AOE + 1.5倍伤害
     - combo=2: 3人合笔（三人合力）
       → 五色晕染全屏 → 大范围AOE + 2.5倍伤害 + 全屏涂色
   - 播放联携特效
   - 全屏墨汁爆炸
   - 重置qteWindow

3. QTE失败
   - qteTimer归零未点击 → qteCombo = 0
   - 正常切换，无联携

4. 合笔按钮UI
   - 水墨圆章造型
   - 出现时有"啪"的水墨扩散动画
   - 倒计时用圆弧进度条表示
   - 只在切换后出现0.8秒

5. 联携特效
   - 2人合笔：两道墨流从左右汇聚到中心→爆炸
   - 3人合笔：三道墨流+五色晕染→全屏爆发
   - 对范围内所有敌人造成伤害
   - 大范围涂色画布覆盖
```

## 四、三项功能的耦合关系

```
战斗中的完整操作流：

1. 走位 → 踩墨池换色 → 针对敌人弱点色攻击
2. 当前画灵怒气满 → 释放爆发技能
3. 想换人 → 点击右侧头像 → 0.8秒QTE窗口
4. QTE成功 → 墨韵联携（2人/3人合笔）→ 大范围伤害
5. 切换后继续 → 新画灵的攻击颜色可能不同 → 又需要踩墨池换色
6. BOSS僵直窗口 → 3人合笔 → 终极输出

三项功能形成闭环：
  踩圈切色（策略层）+ 3人切换（操作层）+ QTE联携（爆发层）
  = 从"站桩输出"变成"策略+操作+时机的完整战斗体验"
```
