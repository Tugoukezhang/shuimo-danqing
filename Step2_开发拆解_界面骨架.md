## 需要新增的界面

按优先级排列，P0必做，P1推荐，P2锦上添花：

### 1. 主界面/大厅 (P0) — 核心中枢
**原版参考**：底部Tab导航（战斗/角色/抽卡/商店/其他），顶部资源条
**我们设计**：
- 顶部：资源条（墨玉/墨晶/灵墨 + 共鸣等级）
- 中央：水墨风场景展示（画卷+当前画灵）
- 底部导航栏：5个Tab
  - 🗺️ 探卷（=关卡选择，已有）
  - 👤 画灵（=角色列表+养成入口）
  - 🎲 墨宝阁（=抽卡系统入口）
  - 🏪 文房四宝（=商店）
  - ⚙️ 设置
- 右侧悬浮：日常任务小红点 + 邮件入口

### 2. 画灵详情/养成 (P0) — 核心养成
**原版参考**：角色详情页→等级/技能/装备/突破
**我们设计**：
- 画灵列表 → 点击进入详情
- 详情页：
  - 画灵立绘（圆形头像+色环）
  - 共鸣等级 + 经验条（共享等级，全局升级）
  - 主动技能（1爆发）+ 被动技能（1退场被动）
  - 装备槽位（3个：墨笔/宣纸/印章）
  - 属性面板（HP/ATK/SPD/CRIT/DEF）
  - 升级按钮（消耗灵墨）

### 3. 装备系统 (P1) — 简化圣遗物
**原版参考**：装备共享+主词条+套装效果
**我们设计**：
- 装备类型：墨笔(攻击)/宣纸(防御)/印章(辅助)
- 品质：白/绿/蓝/紫/金
- 主词条随机 + 0-2个副词条
- 套装2件效果
- 装备背包 + 装备详情 + 穿戴

### 4. 抽卡系统 (P1) — 墨宝阁
**原版参考**：UP池/常驻池，50抽保底
**我们设计**：
- 墨宝阁主界面：水墨画卷翻转
- 两个池子：限定墨韵池(UP) + 常驻墨韵池
- 单抽/十连 → 开卷动画 → 画灵/装备/材料
- 保底计数器显示
- 抽卡历史

### 5. 商店系统 (P2) — 文房四宝
**原版参考**：兑换+购买，多种货币
**我们设计**：
- 标签页：墨玉商品 / 墨晶兑换 / 灵墨兑换
- 商品卡片 + 购买按钮 + 库存限制
- 刷新机制（每日/每周）

### 6. 日常任务 (P2)
**原版参考**：每日5分钟清完
**我们设计**：
- 每日任务列表（3-5个）
- 进度条 + 领奖按钮
- 活跃度宝箱（60/80/100分）

### 7. 挂机收益 (P1)
**原版参考**：世界染色进度=挂机收益倍率
**我们设计**：
- 主界面上半部展示：画卷修复进度
- 一键领取挂机收益
- 收益随时间累积，有上限

### 8. 设置 (P2)
- 音量控制
- 画质选项（粒子数量）
- 操作模式（自动/手动锁敌）
- 账号相关

## 屏幕路由总图

```
title (标题)
  ↓
lobby (主界面/大厅) ←──── 核心中枢
  ├─ 探卷 → select (关卡选择)
  │           ↓ heroSelect (画灵选择)
  │           ↓ battle → result → lobby
  ├─ 画灵 → heroList (画灵列表)
  │           ↓ heroDetail (画灵详情/养成)
  │           │     ↓ equipDetail (装备穿戴)
  ├─ 墨宝阁 → gacha (抽卡)
  │           ↓ gachaResult (抽卡结果)
  ├─ 文房四宝 → shop (商店)
  ├─ 设置 → settings
  ├─ 日常 → daily (日常任务)
  └─ 邮件 → mail (暂时占位)
```

## 数据模型 — 资源系统

Step 2必须先建资源数据模型，因为商店/抽卡/养成都依赖它：

```js
const resources = {
  moYu: 0,       // 墨玉（高级货币，≈钻石）
  moJing: 0,     // 墨晶（中级货币，≈金币）
  lingMo: 0,     // 灵墨（低级货币，养成用）
  gachaTicket: 0,// 墨韵券（抽卡券）
};
```
